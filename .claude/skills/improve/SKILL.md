# 改善提案

振り返りを分析し、スキル・指示文の改善を提案する。提案のみ行い、適用はユーザー判断。

## ワークフロー

### 1. 振り返り収集

`docs/_reflection/*.md` を全て読む。引数でフィルタ可能:
- `/improve` → 全件分析
- `/improve {skill-name}` → そのスキル・拡張機能に関連する知見のみ
- `/improve --since YYYYMMDD` → 指定日以降のみ

各セクションの改善ヒント価値:

| セクション | 価値 | 理由 |
|-----------|------|------|
| 愚痴 | 最高 | 本音＝本質的な痛み。表面的な「学び」より深い |
| こういうふうに指示があれば〜 | 高 | プロンプト・スキル設計の直接ヒント |
| 次回に活かせるアクション | 高 | 未実装なら即提案候補 |
| 得られた知見 | 中 | スキルのリファレンスに反映すべき知見 |
| 刺さったフレーズ | 参考 | ユーザーの指示パターンの傾向 |

### 2. パターン抽出

抽出基準:
- 同じテーマが **2回以上** 出現 → パターン確定
- 1回でも「愚痴」で強い不満 or ユーザー明示的要求 → パターンに含める

パターンごとに記録:
- テーマ（1行要約）
- 出典（ファイル名 > セクション > 引用文）
- 出現回数
- 影響範囲（どのスキル・指示に関係するか）

### 3. 既存スキル・指示との照合

抽出したパターンを以下と照合:
- `.claude/skills/**/SKILL.md` + `references/**`
- `.claude/instructions/**/*.md`
- `CLAUDE.md`（プロジェクト + ユーザーレベル）

照合結果を3分類:
- **未対応**: スキルや指示に全く反映されていない → 提案生成
- **部分対応**: 記載はあるが不十分・古い → 強化提案
- **対応済み**: 既に適切に反映されている → スキップ（レポートに記載）

### 4. 提案生成

出力フォーマット: [proposal-template.md](./references/proposal-template.md)

保存先: `docs/_reflection/_proposals/YYYYMMDD.md`
会話にはサマリー（提案数・優先度別内訳・上位3件の概要）のみ出力。

## 優先度基準

| 優先度 | 条件 |
|--------|------|
| **高** | 3回以上出現 / 愚痴で強い不満 / ユーザー明示的要求 |
| **中** | 2回出現 / 次回アクションに記載あるが未実装 |
| **低** | 1回のみ / 改善効果が限定的 |

## 提案の品質基準

- 各提案に **出典（どの振り返りのどのセクションか）** を必ず明記
- 変更対象ファイルの **現在の内容を読んでから** 提案を書く（推測で書かない）
- 変更イメージは **diff形式** で具体的に示す
- 「〜すべき」ではなく「〜に変更する」と書く（実行可能な粒度）
