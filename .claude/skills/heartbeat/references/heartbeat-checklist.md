# Heartbeat チェックリスト

ops/coach が Heartbeat 実行時に確認する観点。

## 検出対象

| # | 観点 | 具体例 |
|---|------|--------|
| 1 | 同じエラーの繰り返し | ビルドエラー、型エラー、テスト失敗が2回以上 |
| 2 | 前提条件の欠落 | 環境変数未設定、依存未インストール、ブランチ間違い |
| 3 | 曖昧なプロンプト | 指示が不明確でやり直しが発生 |
| 4 | リスクの高い操作 | force push、本番DBへの直接操作、権限昇格 |
| 5 | 過度な冗長性 | 同じ説明の繰り返し、不要な確認の連発 |
| 6 | 手順の抜け漏れ | チェックリスト未実施、後片付け忘れ |

## Root Cause 6分類

検出した問題は必ず以下の6分類のいずれかに分類する。

| 分類 | 説明 | 例 |
|------|------|-----|
| config | 設定ファイル・パラメータの誤り | tsconfig, eslint, vite.config の設定ミス |
| prompt | プロンプトの曖昧さ・不正確さ | スキル定義の指示が不十分 |
| procedure | 手順・プロセスの問題 | デプロイ手順の順序間違い |
| permission | アクセス権限・認可の不足 | settings.json の allow 不足 |
| tool | ツール・スキル・機能の不足 | 必要なMCPサーバーが未設定 |
| external | 外部システム・依存関係の問題 | APIレート制限、ネットワーク障害 |

## 判定基準

- 該当する問題が **1つでも** あれば → improvements.md に記録
- 該当なし → `HEARTBEAT_OK`
- 複数該当する場合 → **最もインパクトが大きい1件のみ** を選択
