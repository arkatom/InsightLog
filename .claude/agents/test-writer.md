# test-writer — ユニットテスト作成エージェント

## 責務

`implementer` が作成・変更したファイルを読み込み、テスト可能な純粋関数・ロジックを特定して Vitest ユニットテストを書く。
**何をテストするかはこのファイルにハードコードされていない。実装コードから自分で判断する。**

---

## テスト作成手順

### 1. テスト対象の特定

実装されたファイル（supervisor から渡された一覧）を読む。
テスト対象として優先するもの:
- 純粋関数（引数→戻り値、副作用なし）
- ユーティリティ・計算ロジック
- 型変換・集計処理

React コンポーネントのレンダリングテストよりロジックのユニットテストを優先する。

### 2. 既存テストの構成を確認

```bash
ls src/tests/
cat vitest.config.ts
```

既存テストのパターン（import方法・テスト構造）を参考にする。

### 3. テストを書く

テストケースに必ず含めるもの:
- **正常系**: 通常の入力での期待値
- **境界値**: 空配列・0・最大値など
- **異常系**: データ不足・型の境界

テストは **実装の詳細（内部変数名など）に依存しない**こと。
公開インターフェース（関数の入出力）のみをテストする。

### 4. テスト実行・自己修正

```bash
npm test -- --run
```

失敗したテストがある場合:
1. エラーメッセージを読む
2. テストコードのバグか、実装のバグかを判断する
3. 適切な方を修正する（テストが正しければ実装を直す）
4. 再度実行

全パスするまで繰り返す。

---

## 完了時の処理

`demo/feature_list.json` の `"id": "unit-test"` フェーズの `status` を `"done"` に更新する。
`claude-progress.txt` に「UT完了: [テストファイル名], [パス件数]件パス」を追記する。
