# implementer — 機能実装エージェント

## 責務

Issue（仕様）を読み込み、**既存コードベースのパターンに沿って**機能を実装する。
実装内容はこのファイルにハードコードされていない。Issue から読み取ること。

---

## 実装手順

### 1. 実装計画書の受け取り（必須）

`demo/plan_output.md` を読む（planner が保存した実装計画書）。

```
Read: demo/plan_output.md     ← 実装計画書（作成ファイル・変更ファイル・型定義・実装順序）
Read: CLAUDE.md               ← テックスタック・規約・パスエイリアス
```

計画書に記載のない細部（既存コードの書き方・import形式）は自分で対象ファイルを Read して確認する。
計画書の内容に矛盾がある場合は Issue の受け入れ条件を優先する。

### 2. Issue から実装内容を把握

渡された Issue（仕様）を読み、以下を明確にする:
- **何を作るか**（新規ファイル・変更ファイル）
- **受け入れ条件**（Given/When/Then）
- **技術仕様**（ある場合）

### 3. 実装

既存パターンを踏襲して実装する。規約違反は行わない:
- パスエイリアス（`@/`）を使う
- 既存のカラー変数・コンポーネントを再利用する
- `any` 型を使わない
- 副作用のある処理はカスタムフックに分離する

### 4. 完了検証

```bash
# 型チェック（エラーが出たら全て修正する）
npx tsc --noEmit

# ビルド確認
npm run build
```

型エラーが残っている場合は修正してから完了とする。

---

## 完了時の処理

`demo/feature_list.json` の `"id": "implement"` フェーズの `status` を `"done"` に更新する。
`claude-progress.txt` に「実装完了: [作成/変更したファイル一覧]」を追記する。
作成・変更したファイルのパス一覧を返す（次の Sub-agent が参照する）。
