# Feature Request: AI ROI ダッシュボード機能の追加

## 概要

InsightLog に **「AI ROI ダッシュボード」モーダル** を追加する。
AI を活用したタスクとそうでないタスクを比較し、AI による時間削減効果（推定）と活用率をビジュアルで示す。

## 背景

InsightLog はすでにタスクごとに AI ツール利用（Claude / ChatGPT / Copilot 等）を記録している。
しかし現状、「AI を使うとどれだけ生産性が上がっているか」を一目で確認できる画面がない。
このダッシュボードにより、個人・チームへの AI 活用促進の定量的根拠として使えるようになる。

## 受け入れ条件

### ケース1: タスクデータが存在する場合

**Given** AI 使用・未使用のタスクが合計5件以上存在する
**When** ヘッダーの「AI ROI」ボタンをクリックする
**Then** モーダルが開き、以下の4カードが表示される:

| カード | 表示内容 |
|--------|---------|
| 今週のAI活用率 | AI使用タスク数 / 総タスク数 のパーセンテージ（0–100%） |
| 推定時間削減 | AI使用 vs 未使用の平均完了時間の差分から算出した「今週の合計削減分（分）」 |
| 最も効果的なカテゴリ | AI 活用率が最も高いタスクカテゴリ名 |
| AI ROI スコア | 時間削減率をもとにした 0–100 のスコアと評価ラベル（Excellent / Good / Fair / N/A） |

**Then** カテゴリ別 AI 活用率の棒グラフが表示される（Recharts 使用）

### ケース2: データが不足している場合

**Given** タスクが0件、またはAI使用・未使用どちらかのデータがない
**When** モーダルを開く
**Then** 「まだデータがありません。タスクを記録して始めましょう 🚀」のメッセージを表示

## 技術仕様

### 新規ファイル
- `src/lib/roiCalc.ts` — ROI 計算ロジック（純粋関数、副作用なし）
- `src/components/report/RoiDashboard.tsx` — ROI ダッシュボードモーダルコンポーネント
- `src/tests/roiCalc.test.ts` — `roiCalc.ts` の Vitest ユニットテスト（5ケース以上）

### 既存ファイルへの変更
- `src/components/layout/Header.tsx` — 「AI ROI」ボタンを追加
- `src/pages/HomePage.tsx` — RoiDashboard モーダルの状態管理と遅延ロードを追加

### デザイン制約
- 既存の Tailwind CSS カラー変数（`primary-*` / `accent-*`）のみ使用
- AIっぽいネオンカラー・過剰なグラデーションは禁止
- モノトーン基調、Shadcn UI ライクな清潔感あるデザイン
- 既存モーダルコンポーネントのレイアウトパターンを踏襲

### テスト要件
- `roiCalc.ts` の全関数をユニットテストでカバー
- 境界値（タスク0件、AI使用0件、AI未使用0件）のテストを含む
- `npm test` がパスすること
