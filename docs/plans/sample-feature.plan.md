# 振り返りメモのバリデーション機能追加

**想定所要時間:** 10-15分
**難易度:** 低〜中
**前提条件:** InsightLog アプリのベース実装が完了していること

## 機能概要

TaskForm の振り返りメモ（notes）に文字数制限のバリデーションを追加する。ユーザーが長文を入力しすぎることを防ぎ、簡潔な振り返りを促す。

## 実装対象ファイル

* **src/components/task/TaskForm.tsx** - バリデーションロジックの追加

## 実装手順

### 1. 文字数カウンター表示の追加

振り返りメモのtextarea下部に、現在の文字数と上限を表示する。

**実装例:**

```tsx
<div className="text-xs text-primary-400 mt-1">
  {notes.length}/500文字
</div>
```

### 2. バリデーションロジックの追加

`handleSubmit` 関数内（89行目以降）に、以下のバリデーションを追加：

```tsx
// 振り返りメモの文字数チェック
if (notes.length > 500) {
  toast.error('振り返りメモは500文字以内で入力してください');
  return;
}
```

**追加位置:** カテゴリチェック（99-102行）の後

### 3. リアルタイムフィードバック（オプション）

500文字を超えた時点で、文字数カウンターを赤色にする。

```tsx
<div className={`text-xs mt-1 ${notes.length > 500 ? 'text-red-500' : 'text-primary-400'}`}>
  {notes.length}/500文字
</div>
```

## 受け入れ基準

以下のすべての条件を満たすこと：

### 入力制限

* **振り返りメモは500文字以内であること**

### バリデーション

* **振り返りメモが500文字を超える場合、送信時にエラーメッセージを表示すること**
* **500文字を超える場合、フォーム送信を防ぐこと**

### UI/UX

* **文字数カウンターを表示すること**（例：「245/500文字」）
* **500文字を超えた場合、文字数カウンターを赤色で表示すること**

## 検証方法

```bash
npm run dev
```

* [ ] 振り返りメモに501文字を入力して「タスクを記録」ボタンを押すと、エラーメッセージが表示されるか
* [ ] 500文字以内の場合は正常に送信できるか
* [ ] 文字数カウンターが正しく表示されるか
* [ ] 500文字を超えた時点で文字数カウンターが赤色になるか

## 注意点

* 文字数カウントは `notes.length` を使用（全角・半角区別なし）
* エラーメッセージは `toast.error()` を使用して表示
* バリデーションは送信時とリアルタイムの両方で実装
